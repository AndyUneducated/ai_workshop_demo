AI 问答助手项目需求与约束

一、核心需求

1. 员工通过对话界面提问，系统基于公司内部知识库返回准确答案
2. 支持追问和多轮对话
3. 答案必须标注来源文档链接，便于员工核实
4. 首期覆盖技术文档和产品需求文档（约 2800 篇）

二、非功能性需求

- 响应时间：首次回答 ≤ 5 秒
- 准确率目标：≥ 80%（相比上次原型的 40% 大幅提升）
- 并发用户：工作时间峰值约 50 人同时使用
- 数据安全：所有数据不得离开公司阿里云账号，禁止使用第三方 SaaS 向量数据库

三、时间与预算

- 工期：两周（10 个工作日），第一周出 MVP，第二周内测迭代
- 可投入人力：2 名后端 + 1 名前端，全职投入
- 硬件预算：可新增一台 GPU 服务器（已申请 A10 24GB），或使用云端 API
- API 预算：OpenAI 月预算 2000 美元，也可考虑本地部署开源模型节省长期成本

四、已排除的方案

- 纯关键词搜索增强：已在 Confluence 上验证，效果不佳
- 微软 Copilot for M365：公司未使用 M365 全家桶，集成成本过高
- 外购 SaaS 产品（如 Glean）：年费 15 万美元超出预算，且数据安全不合规
