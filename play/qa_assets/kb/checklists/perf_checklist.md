# 性能 / 可观测性 / SLO 测试 checklist

适用范围: 所有对外接口 (Web / Mobile / API).

## SLO 标准 (公司级 baseline)

| 接口类别 | P50 | P95 | P99 | 错误率 |
|---|---|---|---|---|
| 鉴权类 (login / refresh) | < 100ms | < 200ms | < 500ms | < 0.1% |
| 写操作 (订单 / 支付) | < 200ms | < 500ms | < 1s | < 0.5% |
| 查询类 (列表 / 详情) | < 80ms | < 200ms | < 400ms | < 0.05% |
| 异步任务回调 | N/A | < 5s | < 30s | < 1% |

接口若超 SLO, 测试必须在 PRD 评审前找到 PM / 架构师讨论.

## 必测项

### 基线压测
- [ ] 单实例 RPS 上限 (爬升至错误率 > 1% 或延迟 > P99 SLO)
- [ ] 全链路压测 (含下游数据库 / 缓存 / MQ)
- [ ] 多副本水平扩展线性度 (1 → 4 → 8 实例)

### 热点场景
- [ ] 秒杀 / 抢购类 (10x 平时流量峰值)
- [ ] 节日营销 (3-5x 流量, 持续 30 分钟)
- [ ] 数据库主从切换期间 (5-10s 不可用窗口)

### 异常恢复
- [ ] 下游缓存全挂 → 接口能继续工作 (有 fallback 到 DB 或 graceful degrade)
- [ ] 数据库慢查询 (> 1s) → 不要拖死服务 (有 statement_timeout)
- [ ] 第三方 API 超时 → 熔断后降级 (circuit breaker pattern)

## 历史教训

| 故障 | 教训 |
|---|---|
| 2025-08 大促接口超时雪崩 | DB 连接池上限 = 100, 雪崩时只放 80 给主请求, 20 给监控/降级路径 |
| 2025-09 邮件队列阻塞 | 同步调外部 SDK 是反模式; 任何外部依赖必须异步化 + 队列 |
| 2025-11 密码字段进 trace | OpenTelemetry 默认全字段抓 trace; auth 路径必须显式 attribute_filter 屏蔽 |

## 可观测性必备

- [ ] 关键接口加 metric (RPS / 延迟分位数 / 错误码分布)
- [ ] 错误日志带 trace_id, 可在 Datadog / Honeycomb 关联
- [ ] 关键业务指标加 dashboard (注册成功率 / 支付成功率 / 等)
- [ ] 报警阈值: 错误率 > SLO 持续 5 分钟自动开 incident
