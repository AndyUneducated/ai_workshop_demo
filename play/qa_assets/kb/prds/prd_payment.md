# 支付 / 结算模块 PRD (历史 v2)

订单完成调三方支付网关 (Stripe / 微信 / 支付宝), 处理异步回调, 更新 order 状态. payment_intent: pending → 第三方回调验签 → 状态机推进; 失败重试 3 次指数退避, 仍失败标 manual_review.

## 关键约束

- 回调必须 HMAC-SHA256 验签, 防伪造
- 同 order_id 重复回调幂等
- 3 min 未收到回调起对账
- 第三方金额必须 = 订单金额, 不一致直接拒

## 上线后教训

1. **重放攻击**: 黑客抓包重发支付成功回调, 余额异常增加. 修复: payload 加 nonce + timestamp, 5 min 外拒, 同 nonce 一次性入 Redis.
2. **金额单位**: 微信"分", Stripe"美分", DB 存"元". 1 元被记成 100 元. 修复: 网关 SDK 层统一换算到"元 + decimal", 业务层永远拿元.
3. **退款竞争**: 用户退款流程中再次发起支付, 状态机错乱. 修复: order_id 级分布式锁, 支付 / 退款全在锁内.
