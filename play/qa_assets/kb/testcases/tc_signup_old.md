# 测试用例集: 邮箱注册 (旧版本, 部分用例失败过, 后修复)

## 元信息

- 关联需求: 注册流程 v1 (REQ-SIGNUP-001, 已废弃, 现合并到 v2)
- 测试人: QA-Daisy
- 状态: TC-SIGNUP-003 / 005 / 007 上线前发现 BUG, 修复后才通过
- 上线日期: 2025-06

## 用例

### TC-SIGNUP-001 [P0][functional] 正常注册
- 给定: 未注册邮箱 + 合法密码
- 当: 提交注册表单
- 那么: 创建 unverified 账号, 发送激活邮件, 返回 200

### TC-SIGNUP-002 [P0][functional] 邮箱格式校验
- 给定: 不合法邮箱 (缺 @, 域名空, 含中文等)
- 当: 提交注册
- 那么: 返回 400 "邮箱格式错误", 不创建账号

### TC-SIGNUP-003 [P0][edge] 同邮箱并发注册 ⚠ 历史 BUG
- 给定: 同一邮箱 X, 在 2 个客户端**同时**(< 100ms) 提交注册
- 当: 数据库 unique 约束触发
- 那么: 一个返回 200 (创建成功), 另一个返回 409 "邮箱已注册"
- 历史问题: 初版没在 service 层加分布式锁, 偶尔出现 race condition,
  两个 unverified 记录都被插入 (虽然 DB unique 后续会拒绝, 但 audit log
  显示两次成功). 修复: register service 加 redis SETNX(email, 60s).

### TC-SIGNUP-004 [P1][boundary] 超长邮箱地址
- 给定: 邮箱长度 > 254 字符 (RFC 5321 上限)
- 当: 提交注册
- 那么: 返回 400 "邮箱过长"

### TC-SIGNUP-005 [P1][security] SQL 注入注册 ⚠ 历史 BUG
- 给定: 邮箱字段含 SQL 注入 payload (`' OR '1'='1` 或 `; DROP TABLE users; --`)
- 当: 提交注册
- 那么: ORM 参数化 query 安全转义, 邮箱按字面字符串处理, 返回 400 (格式校验拒绝)
- 历史问题: 初版用 string 拼接 SQL 写 audit log, 注入到日志库. 修复: ORM 全栈
  + 任何拼接 SQL 的 DAL 都禁用 (lint 规则).

### TC-SIGNUP-006 [P1][edge] 激活邮件 24h 过期
- 给定: 用户 24h+1min 后点击激活链接
- 当: 服务端校验 token expiry
- 那么: 返回 410 "链接已过期, 请重新注册或申请重发邮件"

### TC-SIGNUP-007 [P2][edge] 邮件队列阻塞 ⚠ 历史 BUG
- 给定: 邮件服务暂时不可达 (维护 / 限流)
- 当: 用户注册
- 那么: 注册请求接受, 邮件入持久队列, 用户看到"邮件正在发送, 稍候", 队列 worker
  在邮件服务恢复后异步发送, 用户最长等 30 分钟收到邮件
- 历史问题: 初版同步调邮件 SDK, SDK 卡 30s 超时, 注册请求超时返回 500, 用户看
  到"注册失败"但账号其实已创建, 用户重新注册命中 unique 错误就懵了. 修复:
  注册成功立即返回, 邮件走异步队列.

### TC-SIGNUP-008 [P2][a11y] 错误提示无障碍
- 给定: 表单字段错误 (邮箱格式 / 密码强度)
- 当: 屏幕阅读器读取错误提示
- 那么: 错误文本带 ARIA role="alert", 颜色不是唯一信号 (有图标 + 文字)
