# 用户邮箱注册 PRD (示例)

## 背景

新用户首次使用产品时, 通过"邮箱 + 密码"快速注册账户. 不接入第三方 SSO.

## 用户故事

作为新用户, 我希望用一个邮箱 + 一个密码完成注册, 收到验证邮件后激活账户,
之后可以用同一组凭据登录.

## 功能需求

1. **注册表单**
   - 必填: 邮箱, 密码 (至少 8 位, 含至少 1 个字母 + 1 个数字), 确认密码
   - 邮箱格式校验: 客户端 + 服务端双重
   - 同一邮箱已注册时, 返回明确错误"该邮箱已被注册, 请直接登录"
2. **激活邮件**
   - 注册成功立即发送 1 封含一次性激活链接的邮件
   - 链接 24 小时内有效
   - 用户点击激活前, 账户标记为 `unverified`, 不可登录
3. **重发激活邮件**
   - `unverified` 状态下, 用户尝试登录时给出"请先激活"提示, 并提供"重发邮件"按钮
   - 重发频次限制: 每邮箱每 5 分钟 1 次

## 非功能需求

- 注册接口 P95 延迟 < 300ms (并发 100 RPS)
- 密码字段不记录到任何 log / trace / Sentry 事件
- 表单文案需支持 i18n (中 / 英)
- 错误提示需符合 a11y (有 ARIA label, 颜色不是唯一信号)

## 边界 / 异常

- 网络中断后重试: 同一邮箱 5 分钟内重复请求注册, 服务端幂等
- 邮箱内容过长 (> 254 字符): 服务端拒绝, 返回 400
- 密码包含特殊字符 (`<`, `>`, `&`, 中文): 接受, 不做特殊字符限制
- 邮件服务暂时不可达: 注册请求接受, 入队列重试; 用户看到"邮件正在发送, 稍候"

## 数据 schema

```
users (
  id            uuid pk,
  email         varchar(254) unique not null,
  password_hash varchar(255) not null,    -- bcrypt
  status        enum('unverified', 'active', 'disabled') default 'unverified',
  created_at    timestamp default now(),
  activated_at  timestamp null
)
```

## 已知不做项

- 第三方 SSO (Google / 微信等) — v2 计划
- 手机号注册 — v2 计划
- 双因子认证 (2FA) — v3 计划
