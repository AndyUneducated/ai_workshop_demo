# 用户邮箱注册 PRD (示例)

## 用户故事

新用户用"邮箱 + 密码"注册账户, 收到验证邮件激活后用同一组凭据登录. 不接 SSO.

## 功能需求

1. **注册表单**: 必填邮箱 + 密码 (≥ 8 位, 含字母 + 数字) + 确认密码; 邮箱格式客户端 + 服务端双校验; 同邮箱已注册返回明确错误"该邮箱已被注册, 请直接登录".
2. **激活邮件**: 注册成功立即发送 1 封含一次性激活链接 (24h 有效); 激活前账户 `unverified`, 不可登录.
3. **重发激活**: `unverified` 状态登录时给"请先激活"提示 + "重发"按钮; 每邮箱每 5 分钟最多 1 次.

## 非功能需求

- 注册接口 P95 < 300ms (并发 100 RPS)
- 密码字段不进 log / trace / Sentry
- 表单文案 i18n (中 / 英)
- 错误提示 a11y (ARIA label, 颜色不是唯一信号)

## 边界 / 异常

- 同邮箱 5 分钟内重复请求注册: 服务端幂等
- 邮箱长度 > 254 字符: 返回 400
- 邮件服务暂不可达: 注册请求接受, 入队列重试; 用户看"邮件正在发送, 稍候"
